# Stock Inventory Management System

An API System that can handle all stock management for our clients
We are going to highlight all the requirements our API system is going to have

### Functional
- Create a new order for their stock
- View an order
- Reserve inventory
- Process payment
- Send notification
- Cancel failed orders

### Non-Functional
- Services communicate asynchronously
- Services are independently deployable
- Duplicate events should not break the system
- Failures should be recoverable
- Easy to scale horizontally
- Very Secure
